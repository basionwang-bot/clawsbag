## 一、项目概述

**产品名称**：ClawsBag  
**域名**：clawsbag.com  
**核心定位**：Agent 技能包市场 —— 让普通人的 Agent 真正有用  
**目标用户**：非技术用户（小白），已拥有 OpenClaw / Coze / 扣子等 Agent 工具，但不知道怎么让它解决自己的实际问题  
**核心价值主张**：你的 Agent 缺的不是智能，是真正能用的场景。来这里，找到你行业的技能包，一键装进你的 Agent。

---

## 二、技术栈要求

- **前端**：React + Vite + TailwindCSS
- **后端**：Node.js + Express（或 Next.js 全栈）
- **数据库**：MySQL 或 PostgreSQL（国内服务器友好）
- **部署**：国内服务器（阿里云 / 腾讯云），支持 PM2 进程管理
- **文件存储**：阿里云 OSS（技能包文件上传下载）
- **用户认证**：JWT + 手机号验证码登录（国内用户首选，无需邮箱）

---

## 三、页面结构

```
/ 首页（Landing Page）
/explore 探索/浏览技能包
/skill/:id 技能包详情页
/case 案例中心（PG万联动）
/case/:id 案例详情页
/user/login 登录/注册
/user/profile 个人中心
/user/installed 我已安装的包
/publish 上传/发布技能包（供给方入口）
/admin 后台管理（简单版）
```

### 模块 1：首页 Landing Page

**目标**：让小白用户 3 秒内知道"这是什么、对我有什么用、我怎么开始"

**页面内容（从上到下）**：

#### 导航栏
- Logo：ClawsBag（左上）
- 导航链接：场景库 / 案例 / 发布技能包
- 右上角：登录按钮 / 已登录显示头像

#### Hero 区
- 主标题（大字）：你的 Agent 缺的不是智能，是真正能用的场景
- 副标题：找到你的行业，一键安装现成的技能包和工作流。不需要懂技术。
- 主按钮：浏览场景库（跳转 /explore）
- 次按钮：我要上传技能包（跳转 /publish）
- 背景：深色系，网格纹理，黄色强调色

#### 数据展示条
- 技能包数量 / 覆盖行业 / 安装次数（数字动画效果）

#### 行业入口区（核心）
- 标题："你是做什么的？"
- 行业 Tab 切换：教培培训 / 电商运营 / 自媒体内容 / 销售客服 / 设计创意 / 财务咨询 / 更多
- 每个行业下展示 3 个推荐技能包卡片
- 每张卡片包含：技能包名称、一句话描述、包含技能数、安装次数、一键安装按钮

#### 使用步骤区
- 三步走：① 选场景 → ② 一键安装 → ③ 直接用
- 图文并排，简洁

#### 案例区（PG万联动）
- 标题："别人怎么用的"
- 展示 2-3 个真实用户案例卡片
- 每个案例：头像 + 行业 + 一句话故事 + 核心数据（省了多少时间）
- 底部按钮：查看更多案例 → 跳转 /case

#### 底部 CTA
- "你的 Agent 还缺什么？"
- 大按钮：进入场景库

#### Footer
- Logo + 版权 + 联系方式 + 上传技能包入口

### 模块 2：探索页 /explore

**目标**：让用户快速找到适合自己的技能包

**布局**：
- 左侧：筛选栏（固定）
  - 行业分类（多选）
  - 类型：技能包 / 工作流 / 资源包
  - 价格：免费 / 付费 / 全部
  - 排序：最多安装 / 最新上传 / 评分最高
- 右侧：技能包列表（网格，3列）

**技能包卡片内容**：
- 封面图/图标
- 包名称
- 一句话描述（不超过30字）
- 标签（行业 + 类型）
- 安装次数
- 评分（5星）
- 价格（免费/¥XX）
- 一键安装按钮

**搜索栏**：顶部关键词搜索，支持按名称/描述/标签搜索

### 模块 3：技能包详情页 /skill/:id

**目标**：让用户看完就想安装，清楚知道安装后能做什么

**页面内容**：

#### 顶部信息区
- 技能包名称（大字）
- 发布者信息（头像 + 名称）
- 标签（行业/类型）
- 评分 + 安装次数 + 更新时间
- **核心按钮：一键安装到我的 Agent**（黄色大按钮，最显眼）
- 价格（免费则直接显示"免费"）

#### 解决什么问题（必填）
- "装上这个，你的 Agent 能帮你做什么"
- 用场景语言写，不用技术词汇
- 例："上传一节课录音 → 15分钟内得到学生情绪报告 + 家长沟通话术"

#### 包含内容
- 技能列表（几个skill，每个一句话说明）
- 工作流列表（几个workflow，输入→输出说明）
- 配套资源（提示词模板/知识库等）

#### 使用说明
- 安装后如何使用（步骤，3步以内）
- 适合谁（目标用户描述）
- 不适合谁（使用边界）

#### 用户评价
- 评分 + 文字评价列表
- 支持留评价（需登录）

#### 相关推荐
- 同行业其他技能包（横向滚动卡片）

### 模块 4：一键安装功能

**这是平台核心交互，需要认真设计**

#### 安装流程（用户视角）：

```
用户点击"一键安装"
↓
未登录 → 弹出登录框（手机号验证码）
已登录 ↓
弹出安装弹窗：
  - 显示包内容摘要
  - 选择安装到哪个 Agent（下拉，已绑定的Agent列表）
  - 未绑定Agent → 引导填写 Agent 接入地址/Token
  - 免费包：直接点"确认安装"
  - 付费包：跳转支付 → 支付完成后安装
↓
安装中（loading动画，"正在配置你的Agent..."）
↓
安装成功弹窗：
  - "已成功装入你的 Agent！"
  - 显示安装内容摘要
  - 按钮：去我的 Agent 试试 / 继续浏览
```

#### 技术实现说明（给 Cursor）：
- 安装本质是：将技能包的配置 JSON 写入用户的 Agent 配置
- 第一版可以简化为：生成一个配置文件，提供下载 + 复制，并给出导入步骤说明
- 后续升级为：通过 OpenClaw API 自动写入（等 API 接口确认后对接）
- 用户已安装的包存储在数据库 user_installations 表

### 模块 5：案例中心 /case（PG万联动）

**目标**：用真实故事激发用户"我也能这样"的欲望

**案例列表页**：
- 筛选：行业 / 使用的技能包类型
- 卡片内容：用户头像 + 行业标签 + 标题（一句话故事）+ 核心数据（省X小时/提升X%）+ 用了哪个技能包

**案例详情页 /case/:id**：
- 用户信息（头像/名称/行业/城市）
- 故事正文（第一人称叙述，300-500字）
- 使用了哪些技能包（带跳转链接）
- 使用前/使用后对比（数据卡片）
- 底部：立即安装同款技能包（按钮）

**内容来源**：
- 初期：Basion 自己写（教培案例）
- 后期：用户提交 → 审核后发布

---

### 模块 6：用户注册/登录

**登录方式**：手机号 + 短信验证码（首选，无需密码）  
**备用**：微信扫码登录（后期接入）

**注册流程**：
1. 输入手机号
2. 发送验证码（60秒倒计时）
3. 输入验证码
4. 首次登录需填写：昵称 + 行业（下拉选择）
5. 完成 → 进入个人中心

**个人中心 /user/profile**：
- 基本信息：头像/昵称/行业/简介
- 已安装的技能包列表（跳转 /user/installed）
- 我发布的技能包（如果是供给方）
- 我的评价

**我已安装 /user/installed**：
- 卡片展示所有已安装包
- 每张卡片：包名 + 安装日期 + 操作（重新安装/卸载/查看说明）

---

### 模块 7：上传/发布技能包 /publish

**目标**：让有能力的人轻松上传自己的技能包

**发布表单（分步骤）**：

**步骤一：基本信息**
- 包名称（必填，30字以内）
- 一句话描述（必填，50字以内）
- 行业分类（必选）
- 包类型：技能包 / 工作流 / 资源包 / 组合包
- 封面图（上传，建议尺寸说明）

**步骤二：内容详情**
- 解决什么问题（富文本，必填）
- 包含什么（列表，可动态添加）
- 使用说明（富文本）
- 适合谁/不适合谁

**步骤三：上传文件**
- 上传技能包文件（JSON / ZIP）
- 提示词模板文本（可选，文本框）
- 演示视频链接（可选）

**步骤四：定价**
- 免费 / 付费（填入价格）
- 付费需填写收款信息（微信/支付宝收款码）

**提交后**：进入审核状态，审核通过后上线（初期 Basion 自己审核）

## 五、数据库表结构（参考）

```sql
-- 用户表
users (id, phone, nickname, industry, avatar, created_at)

-- 技能包表
skill_packs (id, title, description, industry, type, price, file_url, 
             author_id, install_count, rating, status, created_at)

-- 技能包标签
skill_tags (id, skill_pack_id, tag_name)

-- 用户安装记录
user_installations (id, user_id, skill_pack_id, installed_at, status)

-- 案例表
cases (id, user_id, title, content, industry, skill_pack_ids, 
       metric_label, metric_value, status, created_at)

-- 评价表
reviews (id, user_id, skill_pack_id, rating, content, created_at)
```

## 六、设计规范

**色调**：深色主题（背景 #0a0a0b）+ 黄色强调色（#f0c93a）
**字体**：
- 标题：Noto Serif SC（有厚重感）
- 正文：Noto Sans SC 300
- 数字/代码：DM Mono

**设计原则**：
- 所有功能按钮用"结果语言"而非技术语言（例："一键安装"而非"导入配置"）
- 行业 Tab 是最重要的导航，始终可见
- 移动端友好（很多用户用手机浏览）
- 加载状态要有反馈（loading / 成功 / 失败）

## 八、初始数据（种子数据）

上线时需要预置以下数据，否则页面为空：

**技能包（最少5个，来自 Basion 自己的教培场景）**：
1. 学生课堂情绪分析 — 上传录音/文字 → 生成情绪报告
2. 学员流失预警 — 追踪出勤/作业 → 关键节点提醒
3. 个性化作业生成 — 根据学员弱点 → 生成差异化练习
4. 家长沟通话术生成 — 输入课堂表现 → 输出沟通文案
5. 课程销售跟进 — 潜客记录 → 自动跟进提醒

**案例（最少1个）**：
- Basion 自己使用"课堂情绪分析"包的真实故事
